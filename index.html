Help & Notes

What it does

  • Generates a 3-column Search table ‚Äî Campaign Name, Adset Name, Keywords ‚Äî for the selected Type.
  • If the Type defines Ads content, a 2-column Ads Copy table ‚Äî Particulars, Ads Copy ‚Äî is also shown.
  • Types can differ. Each Type ships its own variables, sample placeholders, Search rows, and optional Ads rows. In menus and filters you‚Äôll see the internal ID with nickname (e.g. Type 1 (Brand_Appartment)).
  • Inputs are auto-generated from the Type‚Äôs variables list (e.g., BUILDER + PROJECT, CITY) or legacy keywordN usage.
  • Runs fully offline. Templates/logs persist in localStorage; auth state is in sessionStorage.

Placeholders & modifiers

  • Named placeholders (single braces) map 1:1 to inputs: {VAR NAME} like {BUILDER + PROJECT}, {CITY}, {PRICE}.
  • Legacy placeholders (double braces) support optional modifiers: {{keyword1}}, {{keyword2.lower}}, {{keyword3.upper}}, {{keyword4.cap}}, {{keyword5.slug}}.
  • Built-ins (double-brace only): {{today}} (YYYY-MM-DD), {{ts}} (Unix ms).
  • Missing variables resolve to empty strings. Only variable insertions are transformed; static template text is preserved exactly.

Inline JOIN (multi-value helpers)

  • Use {JOIN:VAR A, VAR B, ...} to render only the variables that were provided, joined by / with no spaces. Example: {JOIN:TYPOLOGY 1, TYPOLOGY 2, TYPOLOGY 3} ‚Üí 1/2/3 (if those inputs are present).
  • JOIN is processed before placeholders are transformed, then variable values are Title-Cased (in Ads) and slashes are tightened (no spaces around /).
  • Rows that contain JOIN are hidden unless at least one of the referenced variables has a value (unless X: true is set; see below).

Preview (Search) formatting

  • Only transform variables (not static text) according to column:
    • Campaign Name ‚Üí variables become lowercase_with_underscores.
    • Adset Name ‚Üí variables become lowercase_with_underscores.
    • Keywords ‚Üí variables become lowercase with spaces preserved (no underscores).
  • Static parts from the template remain exactly as written.
  • Search rows are hidden if their required variables are missing (unchanged legacy behavior).

Ads Copy formatting & filtering

  • Only variables are transformed to Title Case; static Ads text is preserved exactly.
  • Spaces around slashes are removed (e.g. 1 / 2 / 3 BHK ‚Üí 1/2/3 BHK).
  • JOIN expressions render as compact slash-lists (e.g., {JOIN:TYPOLOGY 1, TYPOLOGY 2, TYPOLOGY 3} ‚Üí 1/2/3).
  • Row hiding:
    • Ads rows hide if their required variables are missing.
    • Ads rows with JOIN hide if none of the JOINed variables are provided.
    • Ads rows marked X: true always show (override), regardless of missing variables.
  • Dedicated toolbar: Copy All (Ads), Export CSV, Export JSON.

Dynamic sample placeholders

  • Each input shows a sample value from the Type‚Äôs samples (or sampleValues).
  • Samples are hints only ‚Äî change any input to immediately update outputs.

Copy & export

  • Click/tap any cell to copy that cell.
  • Copy All (Search/Ads) copies as TSV (tab-separated) so Sheets/Excel split columns automatically.
  • Exports:
    • Search: CSV / JSON (outputs_search.*)
    • Ads: CSV / JSON (ads_copy.*)
  • Filenames include an ISO timestamp.
  • Shortcuts: Ctrl+S Save log ¬∑ Ctrl+E Export CSV (Search)

Logging, security & template reset

  • Auto-log saves snapshots on changes and key actions (toggle in the Generate form).
  • Idle lock: auto-locks after ~30 minutes of inactivity.
  • Admin-gated: Templates and Logs tabs require the admin password.
  • Always reset templates on refresh: On every page load, shipped defaults overwrite stored templates so no manual reset is needed. You can still import/override during the session.

Prompt: Generate / Update a Type JSON

Use this prompt to create or update a complete Type from one or two source tables (Search and/or Ads). Prefer single-brace variables like {CITY}; use legacy {{keywordN}} only when appropriate. For multi-typology lines, use the inline JOIN helper and/or mark always-visible lines with X: true. 

You are helping me maintain JSON templates for my internal Keyword Generator tool.

    I may share one or both of these tables (Types can differ; include only what exists):
    - Search (3 columns): Campaign Name | Adset Name | Keywords ‚Üí becomes searchRows
    - Ads (2 columns): Particulars | Ads Copy ‚Üí becomes adsRows

    Your tasks:

      {"campaign":"...","adset":"...","keywords":"..."}
    2) Convert each Ads row to:
      {"particular":"...","copy":"..."}  (Optionally add "X": true to force the row to always show)
    3) When a cell shows multiple typologies like "1/2/3/4/5/6 BHK":
      - Replace that run with the inline helper: {JOIN:TYPOLOGY 1, TYPOLOGY 2, TYPOLOGY 3, TYPOLOGY 4, TYPOLOGY 5, TYPOLOGY 6}
      - Keep any trailing words outside JOIN (e.g., "{JOIN:...} BHK Apartments").
    4) Wrap everything into this structure:

    "Type X": {
      "nickname": "Optional_Friendly_Name",
      "columns": ["Campaign Name","Adset Name","Keywords"],   // Search is always 3 columns
      "searchRows": [
        {"campaign":"...","adset":"...","keywords":"..."}
      ],
      "adsColumns": ["Particulars","Ads Copy"],               // Include only if Ads table exists
      "adsRows": [
        {"particular":"...","copy":"..."},
        {"particular":"...","copy":"{JOIN:TYPOLOGY 1, TYPOLOGY 2} BHK", "X": true}  // example with JOIN + always show
      ],
      "variables": ["..."],        // Every unique single-brace label used, e.g. "BUILDER + PROJECT","CITY","TYPOLOGY 1",...
      "samples": {"VAR":"..."}     // A sample value for each variable
    }

    Formatting & rules:
    - Transformations are applied to variables only (not static template text).
    - Search output rules (for variables only):
      ‚Ä¢ Campaign Name ‚Üí lowercase_with_underscores
      ‚Ä¢ Adset Name   ‚Üí lowercase_with_underscores
      ‚Ä¢ Keywords     ‚Üí lowercase with spaces preserved
    - Ads output rules:
      ‚Ä¢ Variables ‚Üí Title Case

      ‚Ä¢ Search rows hide if required variables are missing (default).
      ‚Ä¢ Ads rows hide if required variables are missing or if JOIN has no provided inputs.
      ‚Ä¢ Ads rows with "X": true always show (override).
    - Keep spacing & case in the static text exactly as in source.
    - Use single-brace placeholders like {BUILDER + PROJECT}, {CITY}, {PRICE}. Legacy {{keywordN}} with modifiers is allowed when needed.
    - If a cell is empty, set it to "".
    - Ensure variables[] and samples{} cover all single-brace placeholders used.
    - Return valid JSON only ‚Äî no extra commentary.
