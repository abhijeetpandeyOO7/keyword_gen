Help & Notes

What it does

  • Generates a 3-column Search table ‚Äî Campaign Name, Adset Name, Keywords ‚Äî for the selected Type.
  • If the Type defines Ads content, it is shown as two single-column tables in separate cards:
    • Ads Copy ‚Äî Particulars (1 column: Particulars)
    • Ads Copy ‚Äî Copy (1 column: Ads Copy)
  • Types can differ. Each Type ships its own variables, sample placeholders, Search rows, and optional Ads rows. In menus and filters you‚Äôll see the internal ID with nickname (e.g. Type 1 (Brand_Appartment)).
  • Inputs are auto-generated from the Type‚Äôs variables list (e.g., BUILDER + PROJECT, CITY) or legacy keywordN usage.
  • Runs fully offline. Templates/logs persist in localStorage; auth state is in sessionStorage.

Roles & access

  • Public: access to the Generate tab only.
  • Admin: access to Generate, Templates, Logs, and Help.
  • Use the login popup to choose role (Public/Admin) and enter the corresponding password.

Placeholders & modifiers

  • Named placeholders (single braces) map 1:1 to inputs: {VAR NAME} like {BUILDER + PROJECT}, {CITY}, {PRICE}.
  • Legacy placeholders (double braces) support optional modifiers: {{keyword1}}, {{keyword2.lower}}, {{keyword3.upper}}, {{keyword4.cap}}, {{keyword5.slug}}.
  • Built-ins (double-brace only): {{today}} (YYYY-MM-DD), {{ts}} (Unix ms).
  • Missing variables resolve to empty strings. Only variable insertions are transformed; static template text is preserved exactly.

Inline JOIN (multi-value helpers)

  • Use {JOIN:VAR A, VAR B, ...} to render only the variables that were provided, joined by / with no spaces. Example: {JOIN:TYPOLOGY 1, TYPOLOGY 2, TYPOLOGY 3} ‚Üí 1/2/3.
  • JOIN is processed before placeholders are transformed, then variable values are Title-Cased (in Ads) and slashes are tightened (no spaces around /).
  • Rows that contain JOIN are hidden unless at least one of the referenced variables has a value (unless X: true is set; see below).

Preview (Search) formatting

  • Only transform variables (not static text) according to column:
    • Campaign Name ‚Üí variables become lowercase_with_underscores.
    • Adset Name ‚Üí variables become lowercase_with_underscores.
    • Keywords ‚Üí variables become lowercase with spaces preserved (no underscores).
  • Static parts from the template remain exactly as written.
  • Search rows are hidden if their required variables are missing (legacy behavior).

Ads Copy formatting & filtering

  • Only variables are transformed to Title Case; static Ads text is preserved exactly.
  • Spaces around slashes are removed (e.g. 1 / 2 / 3 BHK ‚Üí 1/2/3 BHK).
  • JOIN expressions render as compact slash-lists (e.g., {JOIN:TYPOLOGY 1, TYPOLOGY 2, TYPOLOGY 3} ‚Üí 1/2/3).
  • Row hiding:
    • Ads rows hide if their required variables are missing.
    • Ads rows with JOIN hide if none of the JOINed variables are provided.
    • Ads rows marked X: true always show (override), regardless of missing variables.

Dynamic sample placeholders

  • Each input shows a sample value from the Type‚Äôs samples (or sampleValues).
  • Samples are hints only ‚Äî change any input to immediately update outputs.

Copy & export

  • Click/tap any cell to copy that cell.
  • Copy All (Search/Ads) copies as TSV so Sheets/Excel split columns automatically.
  • Exports:
    • Search: CSV (outputs_search.*) ‚Äî JSON export is disabled in UI.
    • Ads Copy ‚Äî Particulars: CSV (ads_particulars.*) ‚Äî JSON export hidden.
    • Ads Copy ‚Äî Copy: CSV (ads_copy.*) ‚Äî JSON export hidden.
  • Shortcuts:
    • Ctrl+E ‚Üí Export CSV (Search)
    • Ctrl+Shift+E ‚Üí Export CSV (Ads Copy ‚Äî Particulars)
    • Ctrl+Alt+E ‚Üí Export CSV (Ads Copy ‚Äî Copy)
    • Ctrl+S ‚Üí Save log (manual)
  • Filenames include an ISO timestamp.

Logging, security & template reset

  • Logging runs automatically in the background during edits and actions. (The Auto-log toggle is not shown.)
  • Idle lock: auto-locks after ~30 minutes of inactivity.
  • Admin-gated: Templates and Logs tabs require the admin password.
  • Always reset templates on refresh: On every page load, shipped defaults overwrite stored templates. You can still import/override during the session.

Server-Side Hardening (Hostinger)

You can‚Äôt truly ‚Äúhide‚Äù client-side JS from authenticated users (DevTools/Network will show it). What you can do is block directory browsing, require server auth before any asset loads, add strong security headers, and disable HTML caching so the in-page login always re-prompts.

A) Protect the entire site ‚Äî public_html/.htaccess

Put this file in public_html/.htaccess. Create .htpasswd outside public_html (see section B) and update the path.

# ================================
# Hostinger hardening & auth
# ================================

# 1) Disable directory listing
Options -Indexes

# 2) Force HTTPS
RewriteEngine On
RewriteCond %{HTTPS} !=on
RewriteRule ^ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

# 3) HTTP Basic Auth for entire site
#    IMPORTANT: change to your real home path
AuthType Basic
AuthName "Keyword Generator"
AuthUserFile "/home/YOUR_USER/.htpasswd"
Require valid-user

# 4) Security headers (LiteSpeed/Apache)
<IfModule mod_headers.c>
  Header always set X-Content-Type-Options "nosniff"
  Header always set X-Frame-Options "DENY"
  Header always set Referrer-Policy "no-referrer"
  Header always set Permissions-Policy "geolocation=(), camera=(), microphone=()"
  Header always set X-Robots-Tag "noindex, noarchive, nosnippet, noimageindex"

  # CSP: allow own files; inline is permitted because the app uses it
  Header always set Content-Security-Policy "default-src 'self'; img-src 'self' data:; style-src 'self' 'unsafe-inline'; script-src 'self' 'unsafe-inline'; font-src 'self' data:; frame-ancestors 'none'; base-uri 'self'; form-action 'self'"

  # HSTS (sent only on HTTPS)
  Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" env=HTTPS

  # No-cache for HTML so login gate appears on refresh
  <FilesMatch "\.(?:html?|php)$">
    Header set Cache-Control "no-store, no-cache, must-revalidate, max-age=0"
    Header set Pragma "no-cache"
    Header set Expires "0"
  </FilesMatch>
</IfModule>

# 5) Block sensitive files
<FilesMatch "(^\.|\.env(\..*)?$|composer\.(json|lock)$|package(-lock)?\.json$|yarn\.lock$|pnpm-lock\.yaml$|webpack\.config\.(js|cjs|mjs)$|vite\.config\.(js|ts)$|tsconfig\.json$|.*\.map$|.*\.bak$|.*\.sql$|.*\.log$|robots\.txt$)">
  Require all denied
</FilesMatch>

# 6) Hide VCS & OS junk
RedirectMatch 404 /(?:\.git|\.svn|\.hg|\.DS_Store|Thumbs\.db)(/.*)?$

# 7) MIME defaults
AddDefaultCharset utf-8
AddType text/javascript .js

B) Create .htpasswd (public & admin users)

  • hPanel (no SSH): Security ‚Üí Password Protect Directories. Protect public_html/ (or a subfolder) and add users public and admin. Merge the headers from section A into any .htaccess hPanel creates.
  • SSH:
# Create OUTSIDE public_html
htpasswd -c /home/YOUR_USER/.htpasswd public   # set PUBLIC password
htpasswd    /home/YOUR_USER/.htpasswd admin    # set ADMIN password
  • No SSH: locally run openssl passwd -apr1 "PASSWORD", then paste lines like public:$apr1$...hash... into /home/YOUR_USER/.htpasswd.

C) Optional: protect only /admin area

If you want the app public but lock down /admin:

  1. public_html/.htaccess ‚Äî keep the hardening headers, remove the site-wide Auth* lines.
  2. Create public_html/admin/.htaccess:
# Only admin user can access /admin/*
AuthType Basic
AuthName "Admin Area"
AuthUserFile "/home/YOUR_USER/.htpasswd"
Require user admin

Options -Indexes

D) Build & cache tips

  • Don‚Äôt ship source maps (*.map) to production.
  • Bundle & minify JS/CSS; keep secrets server-side only.
  • If Hostinger cache is enabled, exclude your app path or HTML might still be cached.

After these changes: users must authenticate at the server before any asset loads; directory listing is off; sensitive files are blocked; and HTML won‚Äôt be cached so your in-page login re-prompts on refresh.
